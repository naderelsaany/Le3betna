أهلاً بكم فريق لعبتنا! 🎲
شكراً لمشاركتكم الكود بالكامل، هذا يساعد في تقديم تقييم دقيق وشامل. سأقسم الرد إلى ثلاثة أجزاء كما طلبتم: نقاط القوة، نقاط الضعف والأخطاء، ثم أفضل الممارسات المتقدمة التي ستحدث فرقاً كبيراً في تصدر نتائج البحث.

1. نقاط القوة – ما تم إنجازه بشكل ممتاز

بنية Metadata قوية جداً (layout.tsx): استخدام metadataBase، title.template، openGraph مع siteName، twitter:card، alternates.canonical، verification لـ Search Console، و robots كلها مضبوطة ومتكاملة.

دعم اللغة العربية والاتجاه: lang="ar"، dir="rtl"، وخطوط محملة بطريقة next/font (مما يجعلها ذاتية الاستضافة وأداء ممتاز).

تحسين الصور: استخدام next/image مع fill وsizes ووسوم alt واضحة تحتوي كلمات مفتاحية.

المحتوى النصي غني بالكلمات المفتاحية: هيكلة h1, h2 وعناوين قوية، ونصوص موسّعة تستهدف عبارات مثل "ألعاب لوحية مصرية"، "دومينو اونلاين"، "لودو مجاناً".

بيانات منظمة (Structured Data):

WebSite، WebApplication (مع عرض السعر 0)، و FAQPage، كلها بتنسيق JSON-LD صحيح.

إمكانية وصول جيدة: FAQItem تفاعلي مع aria-expanded و aria-controls.

ملفات أساسية جاهزة: robots.txt يشير إلى sitemap.xml، والخريطة تحتوي روابط المدونة وصفحات ثابتة.

تضمين Google Analytics و Vercel Analytics لمتابعة الأداء.

2. نقاط الضعف والأخطاء الحرجة التي تحتاج تصحيحاً فورياً
أ. تكرار وتضارب Schema من نوع FAQPage

وضعتم FAQPage في layout.tsx (سؤالين عامّين) وأيضاً FAQPage في page.tsx (أربعة أسئلة مخصّصة للصفحة الرئيسية).
المشكلة: وجود كائنين من نفس النوع على نفس الصفحة قد يربك Google، وقد يتجاهل كليهما أو يختار أحدهما دون الآخر، مما يُفقدك فرصة ظهور الـ FAQ الغنية في النتائج.

الحل العاجل:

أزل FAQPage من layout.tsx لأنها ستظهر في كل الصفحات (من نحن، سياسة الخصوصية…) وهذا غير مناسب.

أبقِ فقط FAQPage الخاصة بـ page.tsx على الصفحة الرئيسية.

تأكد أن الأسئلة الأربعة هي نفسها الظاهرة في واجهة المستخدم (وهي كذلك)، فهذا شرط أساسي لظهور النتائج الغنية.

ب. رابط اللعب الرئيسي يخرج من النطاق الأساسي (Link Equity Leak)

جميع أزرار "بدء اللعب" وروابط الألعاب تؤدي إلى https://le3betna-game.vercel.app (نطاق منفصل).
التأثير: تقسيم سلطة الدومين (Domain Authority) وتشتيت قوة الروابط الداخلية، كما أن Google قد تعتبر الموقعين منفصلين تماماً، مما يضعف ترتيب كليهما.

الحل المثالي (ضروري للـ SEO):
استخدم Vercel Rewrites لتوجيه مسار مثل /play أو /game إلى تطبيق الألعاب المنشور على le3betna-game.vercel.app، بحيث يبقى المستخدم (ومحرك البحث) داخل النطاق الرئيسي le3betna.vercel.app.
مثال للإعداد في next.config.js:

js

ج. خريطة الموقع (sitemap.xml) ثابتة وقديمة

الملف public/sitemap.xml يحتوي تواريخ ثابتة (2026-06-27) ولا يُحدّث تلقائياً عند إضافة مقالات جديدة.

الحل: استخدم dynamic sitemap بإنشاء ملف app/sitemap.ts:

ts
Copy
Download
import type { MetadataRoute } from 'next';

export default function sitemap(): MetadataRoute.Sitemap {
  const baseUrl = 'https://le3betna.vercel.app';
  // افترض جلب المقالات من مصدر بيانات
  const blogPosts = [
    { slug: 'how-to-play-domino-online', updatedAt: new Date() },
    // ... إلخ
  ];

  const staticPages = ['', '/about', '/privacy', '/terms', '/blog'].map(route => ({
    url: `${baseUrl}${route}`,
    lastModified: new Date(),
    changeFrequency: route === '' ? 'daily' : 'monthly' as const,
    priority: route === '' ? 1 : 0.8,
  }));

  const blogPages = blogPosts.map(post => ({
    url: `${baseUrl}/blog/${post.slug}`,
    lastModified: post.updatedAt,
    changeFrequency: 'monthly' as const,
    priority: 0.8,
  }));

  return [...staticPages, ...blogPages];
}

هذا يضمن تحديثاً تلقائياً وتواريخ صحيحة.

3. أفضل الممارسات المتقدمة لضمان تصدر النتائج (Advanced SEO)
🧩 تحسين Core Web Vitals والأداء الملموس

تجنب تغيير الـ Layout بسبب الحركات:
في مكوّن FAQItem، استخدم خاصية CSS grid مع grid-template-rows: 0fr و 1fr بدلاً من max-height، لأن الأخيرة قد تسبب إعادة رسم وقد تؤثر على CLS. مثال:

تأكد من تحميل الخطوط بدون وميض: استخدام next/font ممتاز ويمنع FOIT/FOUT.

استخدم priority على الصورة الأولى الظاهرة إذا أضفت صورة فوق الطيّ (hero) لتحسين LCP.

📄 تحسين هيكلة المحتوى للكلمات المفتاحية المستهدفة

أضف ترويسة h1 تحتوي العلامة التجارية والكلمة الرئيسية الأقوى.
h1 الحالي: "العب دومينو ولودو مباشرة من المتصفح". يمكن تحسينه ليصبح:
"لعبتنا: دومينو ولودو وأربعة في صف – ألعاب لوحية مصرية أونلاين"
(مع الحفاظ على العنوان الجذاب، ولكن إدراج "ألعاب لوحية مصرية" في h1 مهم جداً).

استخدم وسوم heading بتسلسل منطقي: حالياً h1 ثم h2 (لماذا تختار…)، h2 (كيفية اللعب)، h2 (اكتشف الألعاب)، h2 (لماذا لعبتنا)، h2 (الأسئلة الشائعة). هذا جيد، لكن تأكد أن كل قسم يتضمن فقرات نصية غنية، وهو ما فعلته في قسم "لماذا لعبتنا".

🔗 بناء شبكة روابط داخلية ذكية

أنشئ صفحة ألعاب فردية لكل لعبة (مثلاً /games/domino) تحتوي على شرح ومراجعة وزر "العب الآن" (الرابط الداخلي /play?game=domino). واربط من الصفحة الرئيسية بتلك الصفحات. هذا يزيد من عمق المحتوى ويستهدف كلمات مثل "تحميل دومينو" أو "قوانين لودو".

أضف BreadcrumbList في صفحات المدونة والألعاب لتعزيز البنية وتسهيل ظهور مسار التنقل في النتائج.

📊 دمج Schema متقدم خاص بالألعاب

على صفحات الألعاب الفردية يمكنك استخدام VideoGame أو Game schema مع خصائص مثل numberOfPlayers, gamePlatform (Web), playMode (Multiplayer), offers. هذا يجعلك مؤهلاً لظهور نتائج منسّقة (Rich Results) إضافية.

أما على الصفحة الرئيسية فيمكنك استخدام ItemList مع عناصر ListItem تحتوي كل منها على Game كمرجع.

📱 تحسين PWA والـ manifest

أنشئ manifest.json يحتوي على name: "لعبتنا", short_name: "لعبتنا", start_url: "/", display: "standalone", أيقونات مختلفة الأحجام. هذا لا يؤثر مباشرةً في الترتيب لكنه يعزز تجربة المستخدم ويزيد من احتمالية "تثبيت" التطبيق مما يرفع إشارات التفاعل.

🔍 تدقيق الزحف والفهرسة

اربط الموقع بـ Google Search Console (لاحظت أنك وضعت كود التحقق بالفعل) وأرسل الـ sitemap الديناميكي الجديد.

استخدم أداة فحص URL لطلب فهرسة الصفحة الرئيسية وأي صفحة جديدة.

تأكد من أن صفحة /blog تظهر قائمة المقالات (ليست فارغة). حاليًا الرابط /blog موجود في السايت ماب؛ تأكد من وجود صفحة مدونة حقيقية تسرد المقالات بروابطها.

🔗 الروابط الخارجية (Backlinks)

ابدأ ببناء روابط خلفية من مواقع ومنتديات ألعاب عربية (مثل مواقع تقييم الألعاب، مدونات التقنية المصرية). ركّز على جودة المحتوى في المدونة ليصبح قابلاً للمشاركة.
