# 🧠 ذاكرة مشروع "لعبتنا" (Project Memory)

**تاريخ التحديث الأخير:** يوليو 2026

## 1. فكرة المشروع الأساسية ووضعه الحالي
- **الهدف:** منصة ألعاب لوحية مصرية أونلاين للمتصفح (دومينو، لودو، أربعة في صف).
- **الوضع الحالي (مكتمل تقنياً 100%):** 
  - صفحة الهبوط `landing_page` اكتملت تماماً ومرفوعة، وتم تطبيق قواعد الـ SEO الصارمة (تحسينات ديب سيك وكيمي)، وإضافة خريطة موقع ديناميكية، و Schema متقدمة، وتحسين الـ H1.
  - تطبيق الألعاب `game_app` مكتمل ومرفوع على Vercel (`le3betna-game.vercel.app`).
  - تم إضافة كود جوجل أناليتكس (`G-LMGQ3RL48H`) في كلا المشروعين بنجاح.
  - تم تفعيل تطبيق الويب (PWA) عبر `manifest.json` في صفحة الهبوط، مما يسمح للزوار بتثبيت اللعبة.

## 2. القيود التقنية والميزانية (Tech & Budget)
- **الميزانية:** صفر (خدمات مجانية بالكامل - Free Tiers - No Credit Card).
- **الاستضافة (Hosting):** Vercel (مربوط بمشروعين: `le3betna` لصفحة الهبوط، و `game_app` للعبة).
- **قواعد البيانات:** Firebase Realtime Database للعب المباشر.

## 3. التقنيات المعتمدة (Tech Stack)
- **إطار العمل:** Next.js 16 (App Router) مع React 19.
- **الخوادم والتزامن:** Firebase Realtime Database مع استخدام `runTransaction` لضمان عدم حدوث تداخل (Race Conditions).
- **التصميم:** Tailwind CSS v4 + Framer Motion + `shadcn/ui`.
- **الفحص الآلي (QA):** استخدام **Playwright** لمحاكاة المستخدمين و **k6** لاختبارات الضغط.

## 4. هيكل المستودع (Repository Structure)
- **المستودع الرئيسي على جيت هب:** `https://github.com/naderelsaany/Le3betna`
- يحتوي حالياً على:
  - `landing_page/` (صفحة الهبوط - Next.js).
  - `game_app/` (منصة اللعب الفعلية - Next.js).
  - `docs/` & `.agents/` (أدوات وملفات المشروع المساعدة).

## 5. الملاحظات والقرارات المستمرة (مُفعلة)
- **القرار 1:** لا استخدام للـ Vercel Rewrites لدمج اللعبة داخل صفحة الهبوط (لأنها تتطلب تعديلات معقدة في كود الـ Game Engine لتعمل من مسار فرعي `/play`). الاعتماد على توجيه المستخدم لـ Subdomain مباشر لضمان استقرار اللعبة.
- **القرار 2:** اعتماد (Next.js + Firebase + Playwright) للوصول لتجربة خالية من الثغرات.
- **القرار 3 (نظام الغرف):** غرف لعب بكود (4 أرقام).
- **القرار 4 (فحص الجودة التلقائي - E2E):** لا يتم اعتبار المشروع منتهياً إلا بعد تشغيل سكريبتات `Playwright` لمحاكاة الجيم كاملاً.
