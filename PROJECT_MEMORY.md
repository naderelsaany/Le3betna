# 🧠 ذاكرة مشروع "لعبتنا" (Project Memory)

**تاريخ التحديث الأخير:** يوليو 2026

## 1. فكرة المشروع الأساسية ووضعه الحالي
- **الهدف:** منصة ألعاب لوحية عربية للمتصفح (دومينو، لودو، أربعة في صف).
- **الوضع الحالي (مكتمل تقنياً):** تم الانتهاء من بناء نظام الألعاب بالكامل، وتم اختباره آلياً.
- **التكوين:** المشروع مقسم إلى جزأين أساسيين داخل مستودع واحد:
  1. `landing_page`: صفحة الهبوط (تعمل حالياً ومرفوعة على Vercel).
  2. `game_app`: تطبيق الألعاب الرئيسي (تم الانتهاء منه وجاهز للرفع).

## 2. القيود التقنية والميزانية (Tech & Budget)
- **الميزانية:** صفر (خدمات مجانية بالكامل - Free Tiers - No Credit Card).
- **الاستضافة (Hosting):** Vercel (مربوط بمشروعين: `le3betna` لصفحة الهبوط، و `game_app` للعبة).
- **قواعد البيانات:** Firebase Realtime Database للعب المباشر.

## 3. التقنيات المعتمدة (Tech Stack)
- **إطار العمل:** Next.js 15 (App Router) مع React 19.
- **الخوادم والتزامن:** Firebase Realtime Database مع استخدام `runTransaction` لضمان عدم حدوث تداخل (Race Conditions) في الأدوار.
- **التصميم:** Tailwind CSS v4 + Framer Motion (للأنيميشن السلس) + `shadcn/ui`.
- **الفحص الآلي (QA):** تم تثبيت مهارة فحص الجودة الإجبارية. نستخدم **Playwright** لمحاكاة المستخدمين و **k6** لاختبارات الضغط (إن تطلب الأمر).
- **مراجعة الكود:** تم ربط سير العمل بـ **CodeRabbit** (AI Code Reviewer) لمراجعة الأكواد تلقائياً على GitHub.

## 4. هيكل المستودع (Repository Structure)
- **المستودع الرئيسي على جيت هب:** `https://github.com/naderelsaany/Le3betna`
- يحتوي حالياً على:
  - مجلد `.agents/` و `docs/`.
  - مجلد `landing_page/` (صفحة الهبوط).
  - مجلد `game_app/` (منصة اللعب الفعلية التي تم بناؤها).

## 5. الملاحظات والقرارات المستمرة (مُفعلة)
- **القرار 1:** استخدام الباقات المجانية فقط (Firebase + Vercel).
- **القرار 2:** اعتماد (Next.js + Firebase + Playwright) للوصول لتجربة "Premium" وخالية من الثغرات.
- **القرار 3:** الدخول للتطبيق عبر (Google Authentication) كخيار أساسي، مع دعم الدخول كزائر (Anonymous) لتسهيل التجربة السريعة.
- **القرار 4 (نظام الغرف):**
  - غرف لعب بكود (4 أرقام).
  - تعتمد الألعاب (مثل Ludo و Connect 4 و Domino) على نظام "النسخ المتزامنة - Versioning" في الفايربيز لمنع فقدان البيانات.
- **القرار 5 (فحص الجودة التلقائي - E2E):** لا يتم رفع أي كود رئيسي إلا بعد تشغيل سكريبتات `Playwright` لمحاكاة الجيم كاملاً (فتح متصفحين، الانضمام لغرفة، لعب أدوار) لضمان متانة الكود.

## 6. الخطوات القادمة (Next Steps)

- استدعاء `CodeRabbit` لأول مراجعة شاملة للعبة.
- رفع `game_app` كـ Project جديد على Vercel ليكون متاحاً للجمهور (Production).
