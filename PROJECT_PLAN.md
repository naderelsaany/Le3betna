# 📋 خطة مشروع "لعبتنا" الشاملة (Comprehensive Project Plan)

هذا الملف هو الخريطة الكاملة للمشروع، ويحتوي على كل التفاصيل التقنية، المهام، والقواعد الأساسية التي تم الاتفاق عليها.

---

## 🎯 قواعد المشروع الأساسية
1. **لا للأزرار الوهمية (No Dummy UI):** أي زر في الواجهة يجب أن يعمل بالكامل.
2. **مجانية بالكامل (Free Tiers):** الاعتماد فقط على الباقات المجانية لـ Vercel و Firebase.
3. **منهجية العمل (Game-First):** برمجة منطق اللعبة واختبارها أونلاين أولاً، ثم بناء باقي التطبيق حولها.
4. **التصميم (Premium & Mobile-First):** دعم كامل للـ Dark/Light mode بتناسق ألوان احترافي، وتجربة مستخدم مريحة وممتازة على الموبايل.

---

## 🏗️ المرحلة 1: التأسيس والبنية التحتية للمشروع (Setup)
- [x] إنشاء فولدر `game_app` وتسطيب Next.js 16 (App Router) بدلاً من 15.
- [x] تسطيب Tailwind CSS v4, Framer Motion, shadcn/ui مع مراعاة نظام (CSS-first) الجديد.
- [x] تهيئة Firebase وربطه بالمشروع، وكتابة (Security Rules) صارمة من اليوم الأول لمنع الغش.
- [x] ضبط الـ Design System للـ Dark Mode.
- [x] تفعيل الدخول المجهول (Anonymous Auth) مؤقتاً لتوليد UID ثابت أثناء اختبار اللعبة قبل ربط جوجل.

---

## 🎲 المرحلة 2: محرك اللعبة الأساسي ونظام الغرف (التركيز على لعبة أربعة في صف كـ MVP)
*سنبدأ ببرمجة أول لعبة كصفحة اختبار معزولة للتأكد من كفاءة اللعب الأونلاين.*
- **منطق الغرف (Room Logic):**
  - [x] برمجة نظام إنشاء الغرفة بكود 4 أرقام فقط (مع فحص الداتا بيز أولاً للتأكد من عدم تكرار الكود النشط).
  - [x] مزامنة حالة اللاعبين باستخدام (onDisconnect) و (Heartbeat) لمعرفة متى يغلق اللاعب المتصفح.
  - [x] حذف الغرفة تلقائياً إذا انقطع اتصال اللاعبين لضمان عدم استهلاك مساحة الداتا بيز.
- **لعبة "أربعة في صف - Connect 4" (اللعبة الأولى):**
  - [x] فصل منطق اللعبة (Game Engine) في كود نقي بعيداً عن Firebase لسهولة اختباره.
  - [x] استخدام (Firebase Transactions) لكتابة الحركات لمنع تداخل اللعب (Race Conditions).
  - [x] استخدام Zustand لإدارة حالة الواجهة فقط (UI State)، بينما Firebase هو المصدر الوحيد لحالة اللعبة.
- **الشات (In-Game Chat):**
  - [x] برمجة شات سريع داخل الغرفة أثناء اللعب.

## 🧪 المرحلة 2.5: الاختبارات (Testing & Anti-Cheat)

## 🌟 إضافات مقترحة لرفع مستوى التطبيق (Recommendations)
*(سيتم إدراجها في خطة التنفيذ في حال موافقتك)*
1. **نظام تقييم ونقاط (Leaderboard / Ranking):** يحصل اللاعب على نقاط عند الفوز ونخسرها عند الهزيمة، لعمل حافز وتنافس بين اللاعبين.
2. **حالة الاتصال (Online Presence):** عرض نقطة خضراء بجانب اللاعبين النشطين (Online) لمعرفة من متاح للعب.
3. **مؤثرات صوتية واهتزازات (Sound & Haptics):** تشغيل صوت عند تحريك القطعة أو عند الفوز لزيادة المتعة (Game Feel).
4. **تاريخ المواجهات (Match History):** تسجيل سريع لآخر 5 مباريات لعبها المستخدم في حسابه.
